# V2.84

Date: 2026-07-24

## 本輪主題

去背用純色背景從「只有綠色」擴充成「綠／藍／洋紅三選一」，並改用使用者提供的更精簡咒語寫法。版本升至 v2.8.4。

## 背景

v2.8.3 把去背底改回純綠 `#00FF00`。使用者接著補充完整的專業去背色選建議：

1. **純綠 `#00FF00`**（首選）——幾乎沒有人類膚色、很少出現在一般服裝、AI 去背辨識率最高、Photoshop/Premiere 都支援。
2. **純藍 `#0000FF`**（次選）——當角色設計本身帶大量綠色（草地、綠色衣服、翡翠、精靈系角色）時，綠幕會把角色一起去背，這時要改用藍幕。
3. **亮洋紅 `#FF00FF`**（第三選）——角色綠、藍都有時的備案，很多專業 CG 也用這個顏色，人像幾乎不會出現。

這是標準的去背色選邏輯：任何一種 Chroma Key 顏色都可能跟角色本身的設計「撞色」，所以要留備案，而不是只認一種顏色。

## 本輪完成

- `bgData` 新增 `blue`（`#0000FF`）、`magenta`（`#FF00FF`）兩個條目，並抽出共用函式 `chromaKeyBg(hex, colorLabel, whyThisColor)`，三色共用同一套「無漸層/無陰影/無紋理/無光影變化/完全獨立背景」規則文字，只是色號與「為什麼選這個顏色」的說明不同。
- 咒語措辭改用使用者提供的更精簡版本：`Solid chroma key background, flat pure ${color} (${hex}), uniform color, no gradients, no shadows, no texture, no lighting variation, completely isolated background.` 開頭，取代先前偏長的舊版描述。
- 背景區塊 UI 從 3 個選項擴充到 **5 個**：🟢純綠去背底（推薦，預設）／🔵純藍去背底／🟣亮洋紅去背底／⚪單純背景／🎨有背景豐富元素，每個選項的說明文字都講清楚「什麼情況該選這個」（例如藍底標註「角色身上有大量綠色時改用這個」）。
- `selectBg()`／`restoreWorkspace()` 白名單同步擴充到 5 個合法值。

## 沒有動的部分

`003圖片暫存區/process.mjs` 去背腳本本身邏輯不用改——它是自動從裁切邊框取樣背景色，不管使用者選綠、藍、洋紅、白，都能正確偵測到那格實際用的顏色再去背，完全通用。
