# V2.70

Date: 2026-07-22

## 本輪主題

V4：把三種創作方案（人物+文字／人物+情緒特效／純文字主體）合一成同一個工具，可自由混搭；並補齊 LINE 上架必備的主圖／標籤圖 prompt，背景預設改青綠色方便去背。版本升至 v2.7.0。

## 背景

上一輪提案討論出三個獨立方向（方案 A/B/C），使用者的結論是：不要三選一，要合一，並且可以在同一組 16 張貼圖裡混搭（例如 12 張走純文字、4 張走情緒特效）。

## 本輪完成

### 3 模式合一（核心）

- 新增「內容模式」切換：👤💬 人物＋文字（預設，等同舊行為）／👤✨ 人物＋情緒特效／🔤 純文字主體。
- 用行首方括號標籤（`[情緒]`／`[文字]`）讓現有的「16 行文字框直接編輯」流程可以混搭三種模式，不用重做整個編輯體驗：
  - 未標記的行永遠是人物+文字模式，向後相容，沒用過新功能的人 100% 無感。
  - 從情緒卡／純文字挑選器點選插入時自動加標籤，凍結在挑選當下的模式，不受之後切換全域模式影響。
- 新增 24 張情緒卡（`EMOTION_CARDS`）：每張含表情動作＋背景特效描述，供「人物＋情緒特效」模式挑選。
- `combineAll()` 依標籤分流，各自產生對應的 Panel prompt：人物+文字沿用原邏輯不變、情緒特效輸出「EMOTION+FX PANEL」、純文字主體輸出「TEXT-HERO PANEL」（文字占比反轉為主體 55-70%，角色降為配角）。
- 字體系統（40 套字體風格）跟版面 C（大字置中+角色置角）直接沿用既有資料，不用重建。

### 主圖／標籤圖（上架必備，之前完全沒做）

- 新增「⑥ 封面圖／標籤圖」區塊，各生成一份獨立 Prompt：
  - 主圖 240×240px：角色最有代表性的姿勢/表情。
  - 標籤圖 96×74px：極簡臉部特寫，透明底，無文字。
- 沿用目前選定的角色設定與背景風格，跟 16 張貼圖視覺一致。

### 背景改青綠色去背

- 背景選項新增「青綠去背底」，設為新預設（原本白底/AI背景仍可選）。
- `bgData` 新增 `green` prompt 區塊；`selectBg`／`randomizeAll`／`smartFillWorkspace` 等所有預設值一併更新。

### `003圖片暫存區/process.mjs` 去背邏輯升級（獨立於本 repo，另行變更）

- 原本寫死抓白色當背景色，改成自動從每一格四個角落取樣背景色，用歐氏距離判斷「接近背景色」，白底、青綠底皆適用，不用使用者手動指定。
- 連通區塊分析（外部背景 vs 角色內部同色缺口）邏輯不變，只是把顏色判斷從「是否為白色」換成「是否接近取樣到的 key color」。
- 用一支獨立驗證腳本（不動使用者既有 output 目錄）在真實白底圖片上測試：正確識別出背景色 (254,254,254)，去背後不透明像素從 100% 降到 24.4%，裝飾小物內部的白色（愛心、警示牌、手機圖示）正確保留，無回歸問題。

### 測試

- smoke test 新增 4 組（三模式合一存在性、情緒卡資料完整性、標籤解析正確性、青綠底預設），共 22 項全過。
- 額外用獨立腳本模擬 `combineAll` 混合三種模式標籤的分流輸出，逐行核對正確。

## 已知限制 / 下一輪候選

- 情緒卡 24 張、字體 40 套是初版量，之後可依實際使用回饋調整。
- 主圖/標籤圖目前是獨立 Prompt，需要另外請 AI 生成兩張圖（不是自動合併在 16 張裡）。
- `process.mjs` 的改動不在這個 git repo 管轄範圍內（在 `003圖片暫存區`），需要另外留意版本追蹤。
