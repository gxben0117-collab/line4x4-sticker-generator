# V2.81

Date: 2026-07-24

## 本輪主題

移除「青綠去背底」背景選項，背景只留白底／AI背景兩種。版本升至 v2.8.1。

## 背景

v2.7.0 曾把預設背景改成青綠色（chroma-key），方便去背腳本切割貼圖。使用者評估後決定拿掉這個選項，介面只留：

- 單純背景（乾淨純白背景）
- 有背景豐富元素（根據貼圖主題自動補充情境）

## 本輪完成

- `index.html` 背景區塊拿掉「青綠去背底」`.c-item`，只剩白底／AI背景兩個選項，白底改回預設（`bg-white` 帶 `active`）。
- `state.bgStyle` 預設值 `'green'` → `'white'`；`bgData` 拿掉 `green` 條目；`selectBg()` 的合法 key 清單拿掉 `'green'`。
- `restoreWorkspace()` 對 `payload.bgStyle` 加白名單驗證（只接受 `'white'`/`'ai'`），避免舊 localStorage 存檔殘留的 `'green'` 值造成 `bgData[state.bgStyle]` 讀到 `undefined`、咒語輸出出現字面上的 `undefined`。
- init 區的 fallback `selectBg(state.bgStyle || 'green')` 改回 `'white'`。
- smoke test 把原本斷言「green 存在」的測試改成斷言「green 不存在」（`id="bg-green"`、`selectBg('green')`、`chroma-key green` 三個字串都確認已清除）。

## 驗證

- `npm run build && npm run lint && npm test`：33 項全過。
- 用 puppeteer 實際跑本機伺服器，確認背景區塊只剩 2 個選項、預設選中「單純背景」，畫面沒有跑版，console 只有一個跟這次改動無關的 favicon 404。

## 附註

`003圖片暫存區/process.mjs`（去背切割腳本，獨立於本 repo）目前仍保留「自動偵測背景 key color」的通用去背邏輯（白底、青綠底皆可用），這次沒有改動它——即使咒語工具不再主動推薦青綠底，該腳本仍相容使用者手動要求 AI 生成青綠底或其他任何底色的舊圖。
