# LINE 貼圖 4×4 咒語產生器 - 部署說明

## 線上網址
部署完成後，網站將可在以下網址訪問：
```
https://gxben0117-collab.github.io/line4x4-sticker-generator/
```

## 部署方式
本專案使用 GitHub Pages 進行部署，採用 GitHub Actions 自動化部署流程。

## 部署步驟

### 1. 推送代碼到 GitHub
```bash
cd "C:\AIProjects\002專案進行中\貼圖line4x4咒語產生器"
git add -A
git commit -m "deploy: 準備部署到 GitHub Pages"
git push origin master
```

### 2. 啟用 GitHub Pages
1. 前往 GitHub 倉庫：https://github.com/gxben0117-collab/line4x4-sticker-generator
2. 點擊 **Settings** 標籤
3. 在左側選單找到 **Pages**
4. 在 **Source** 下拉選單中選擇 **GitHub Actions**
5. 等待部署完成（約 1-2 分鐘）

### 3. 驗證部署
部署完成後，訪問：
```
https://gxben0117-collab.github.io/line4x4-sticker-generator/
```

## 自動部署
每次推送到 `master` 分支時，GitHub Actions 會自動觸發部署流程：
1. 檢出代碼
2. 配置 GitHub Pages
3. 上傳網站文件
4. 部署到 GitHub Pages

## 文件結構
```
.
├── index.html              # 主頁面
├── assets/                 # 資源文件
│   ├── styles.css         # 優化後的樣式
│   ├── app.js             # 應用邏輯
│   └── data.js            # 數據配置
├── .github/
│   └── workflows/
│       └── deploy.yml     # GitHub Actions 部署配置
├── .nojekyll              # 禁用 Jekyll 處理
└── README.md              # 專案說明
```

## 注意事項

### 1. 文件路徑
- 所有資源使用相對路徑
- 確保 `index.html` 在根目錄
- CSS 和 JS 文件在 `assets/` 目錄

### 2. 瀏覽器兼容性
- Chrome/Edge: 完全支持
- Firefox: 完全支持
- Safari: 完全支持
- 移動端瀏覽器: 完全支持

### 3. 性能優化
- 單頁應用，無需服務器
- 所有資源本地加載
- 支持離線使用（瀏覽器緩存）

### 4. 更新流程
1. 修改代碼
2. 本地測試
3. 提交並推送到 GitHub
4. 自動部署（約 1-2 分鐘）

## 本地測試
在推送前，可以本地測試：
```bash
# 使用 Node.js serve
npm run dev

# 或直接打開 index.html
start index.html
```

## 故障排除

### 部署失敗
1. 檢查 GitHub Actions 日誌
2. 確認 Pages 權限已啟用
3. 驗證 workflow 文件語法

### 頁面無法訪問
1. 確認 GitHub Pages 已啟用
2. 檢查倉庫是否為 public
3. 等待 DNS 傳播（最多 10 分鐘）

### 樣式或功能異常
1. 清除瀏覽器緩存
2. 檢查控制台錯誤
3. 驗證文件路徑正確

## 自定義域名（可選）
如果要使用自定義域名：
1. 在倉庫根目錄創建 `CNAME` 文件
2. 文件內容為你的域名（例如：`sticker.example.com`）
3. 在域名 DNS 設置中添加 CNAME 記錄指向 `gxben0117-collab.github.io`
