# 🔧 GitHub Pages 404 錯誤排除指南

## 問題現象
訪問 `https://gxben0117-collab.github.io/line4x4-sticker-generator/` 出現 404 錯誤。

## 解決步驟

### 步驟 1：確認 GitHub Pages 設置

1. 前往倉庫：https://github.com/gxben0117-collab/line4x4-sticker-generator

2. 點擊 **Settings** 標籤

3. 在左側選單找到 **Pages**

4. 檢查 **Source** 設置：
   - ✅ 應該選擇：**GitHub Actions**
   - ❌ 如果是 **Deploy from a branch**，請改為 **GitHub Actions**

5. 保存設置後，等待 1-2 分鐘

### 步驟 2：檢查 Actions 權限

1. 在倉庫中，前往 **Settings** > **Actions** > **General**

2. 找到 **Workflow permissions** 區域

3. 確認選擇了：
   - ✅ **Read and write permissions**
   
4. 勾選：
   - ✅ **Allow GitHub Actions to create and approve pull requests**

5. 點擊 **Save** 保存

### 步驟 3：手動觸發部署

1. 前往 **Actions** 標籤

2. 在左側選擇 **Deploy to GitHub Pages**

3. 點擊右側的 **Run workflow** 按鈕

4. 選擇 **master** 分支

5. 點擊綠色的 **Run workflow** 按鈕

6. 等待部署完成（約 1-2 分鐘）

### 步驟 4：檢查部署狀態

1. 在 **Actions** 標籤中，查看最新的 workflow 運行

2. 確認狀態：
   - ✅ 綠色勾勾 = 成功
   - ❌ 紅色叉叉 = 失敗（點擊查看錯誤日誌）

3. 如果成功，等待 1-2 分鐘後重新訪問網站

### 步驟 5：清除瀏覽器緩存

如果以上步驟都完成了，但仍然看到 404：

1. 按 **Ctrl + Shift + R**（Windows）或 **Cmd + Shift + R**（Mac）強制刷新

2. 或者使用無痕模式訪問

3. 或者清除瀏覽器緩存後重試

## 常見問題

### Q1: 為什麼需要選擇 GitHub Actions？
A: 因為我們使用了自定義的 GitHub Actions workflow 來部署，而不是傳統的分支部署方式。

### Q2: 部署需要多久？
A: 通常 1-2 分鐘。第一次部署可能需要 3-5 分鐘。

### Q3: 如何確認部署成功？
A: 在 Actions 標籤中看到綠色勾勾，並且在 Settings > Pages 中看到綠色的 "Your site is live at..." 訊息。

### Q4: 倉庫需要是 Public 嗎？
A: 免費的 GitHub Pages 需要 Public 倉庫。Private 倉庫需要 GitHub Pro。

## 檢查清單

請確認以下所有項目：

- [ ] 倉庫是 Public（或有 GitHub Pro）
- [ ] Settings > Pages > Source 設置為 **GitHub Actions**
- [ ] Settings > Actions > General > Workflow permissions 設置為 **Read and write**
- [ ] Actions 標籤中最新的 workflow 運行成功（綠色勾勾）
- [ ] 等待至少 2-3 分鐘讓 DNS 生效
- [ ] 清除瀏覽器緩存或使用無痕模式

## 如果還是不行

### 方案 A：使用傳統分支部署

1. 在 Settings > Pages > Source 選擇 **Deploy from a branch**
2. Branch 選擇 **master**，文件夾選擇 **/ (root)**
3. 點擊 Save
4. 等待 1-2 分鐘

### 方案 B：檢查文件結構

確認 `index.html` 在倉庫根目錄：
```
line4x4-sticker-generator/
├── index.html          ← 必須在這裡
├── assets/
├── .github/
└── ...
```
