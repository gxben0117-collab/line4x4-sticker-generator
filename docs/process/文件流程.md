# 文件流程

## 新增正式版本時

1. 先在 `index.html` 完成修改
2. 視需要保留新的 HTML 快照到 `versions/`
3. 執行 `npm.cmd run build`
4. 執行 `npm.cmd test`
5. 新增一份開發紀錄到 `docs/development-log/`

## 文件更新原則

- 不覆蓋舊版開發紀錄
- 每次正式變更新增一份 `Vx.xx.md`
- 如果只是零碎討論，不一定要立即升版
- 只有能交付、能描述、能回頭追的變更才寫進正式紀錄

## 建議交付物

- `index.html`
- `dist/index.html`
- 新版開發紀錄
- 如有對外快照，再新增對應 `versions/*.html`
