# LINE 貼圖分類 V3 再分析補充

產生日期：2026-07-09

用途：補充 `line-sticker-taxonomy-v3.md` 已實作後的再檢查意見。這份文件不取代既有 V3 taxonomy，也不直接修改 `index.html`。

## 再分析結論

目前方向正確：V3-lite 已經比原始 86 類更接近使用者思維。

接下來不建議直接做 V4 大重構，而是先做「V3 精修」：

1. 確認首頁入口是否真的以 `🔥 熱門100句` 為第一入口。
2. 確認 `全部` 是否已降為工具入口，而不是主入口。
3. 檢查熱門100句是否真的只有 100 句，且都短、常用、跨情境。
4. 檢查重複句是否只顯示一次，其他分類只保留 tag。
5. 把 7 字以上、英文長句、混合字句列為後續修字清單。

## 產品判斷

使用者不是在想「我要找某個資料分類」，而是在想：

- 我要快速回覆。
- 我要表達情緒。
- 我要說早安晚安。
- 我要做一包工作貼圖。
- 我要做一包戀愛貼圖。
- 我要做無字表情。

所以首頁應該優先呈現使用場景，而不是資料來源。

## 建議首頁順序

1. 🔥 熱門100句
2. ⭐ 常用聊天
3. 😀 情緒反應
4. 🌞 早安晚安
5. ❤️ 戀愛情侶
6. 💼 工作職場
7. 👨‍👩‍👧 家人朋友
8. 🍔 生活娛樂
9. 🐶 寵物
10. 🎉 節日祝福
11. ✨ 無字貼圖
12. 🎨 特殊主題

`全部` 建議保留在搜尋或資料檢視，不放在主入口第一顆。

## 比既有 V3 更需要注意的地方

### 1. 熱門100句應該是產品核心

這不是普通分類，而是最重要的使用入口。

熱門句應符合：

- 2-5 字為主。
- 可跨聊天、工作、日常使用。
- 不依賴特定角色或節日。
- 不含難畫的英文長字。
- 不含過多標點。

高優先句：

- 收到
- OK
- 好
- 好喔
- 早安
- 晚安
- 哈哈哈
- 笑死
- 真的假的
- 加油
- 辛苦了
- 我看看
- 馬上來
- 等等我
- 在路上
- 我到了
- 沒問題
- 謝謝你
- 對不起
- 抱抱

### 2. 特殊主題必須降權

以下分類可以保留，但不應該跟常用聊天同級：

- 名人語錄
- 成語金句
- 三字經名句
- 修仙養生
- 渣男語錄
- 渣女語錄
- 美髮沙龍
- 美髮沙龍專業包

這些適合做特定貼圖包，不適合作為一般使用者找句子的主要入口。

### 3. 生活娛樂不宜過大

生活娛樂會吸收很多分類，容易又變成大垃圾桶。

建議二級分類最多保留：

- 吃喝
- 出遊
- 追劇追星
- 購物
- 運動健康
- 天氣季節
- 3C遊戲
- 金錢理財
- 讀書考試

如果再細分太多，會回到原本 86 類太散的問題。

### 4. 無字貼圖要跟文字貼圖分清楚

`純表情無字`、`純表情·正面積極`、`純表情·旅拍姿勢`、`純旅拍大片·無文字` 裡的句子不是要印字，而是動作描述。

UI 文案應該明確標示：

> 此分類作為表情 / 姿勢 / 動作描述，出圖時可設為無文字。

### 5. 長句與英文長句先不要刪，先標記

先標記，不急著刪。

優先修：

- `Merry Christmas` -> `聖誕快樂`
- `Happy Birthday` -> `生日快樂`
- `Happy New Year` -> `新年快樂`
- `deadline到了` -> `死線到了` 或 `趕稿中`
- `email轟炸` -> `信件爆炸`
- `不要給自己太大壓力` -> `別太有壓力`
- `防曬是最好的保養` -> `防曬要做`
- `不放棄的樣子最美` -> `別放棄`

## 下一步建議

### Step 1：審既有 V3 實作

先確認目前網站是否已做到：

- `🔥 熱門100句` 預設打開。
- `特殊主題` 排最後。
- 舊分類名只作 tag，不干擾首頁入口。
- 熱門句不超過 100 句。

### Step 2：做 V3 草稿差異檢查

檢查：

- `line-sticker-sentences-v3-draft.md`
- `index.html` 的 `HOT_SENTENCES`
- `scriptData` 或分類資料

確認文件、草稿、實際網站是否一致。

### Step 3：只修高影響問題

優先順序：

1. 首頁入口順序。
2. 熱門100句品質。
3. 長句提醒與修字清單。
4. 無字分類文案。
5. 特殊主題降權。

暫時不要做：

- 收藏
- 最近使用
- AI 推薦
- 完整搜尋 tag 系統
- JSON 拆檔大重構

## 結論

目前不是要重做一套分類，而是要檢查已實作的 V3 是否真的落在使用者路徑上。

最值得做的下一件事：

> 對照 `index.html` 目前實作，列出 V3 taxonomy、V3 draft、實際 UI 三者不一致的地方。

確認不一致後，再做小範圍修正。
